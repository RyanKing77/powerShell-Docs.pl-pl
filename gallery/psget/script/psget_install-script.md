# <a name="install-script"></a>Skrypt instalacji

Instaluje pliki skryptów programu PowerShell z repozytoriami online na komputerze lokalnym.

## <a name="description"></a>Opis

Polecenia cmdlet Install-skryptu znajduje i pobiera jeden lub więcej skryptów z galerii online, weryfikuje i instaluje je na komputerze lokalnym do zakresu określonej instalacji.

Jeśli żaden zakres nie jest zdefiniowana lub gdy wartość parametru zakresu jest AllUsers, skrypt zainstalowano %systemdrive%:\Program Files\WindowsPowerShell\scripts. Wartość zakresu po CurrentUser skrypt zainstalowano $home\Documents\WindowsPowerShell\scripts.

Można filtrować wyniki oparte na wersji minimalnej i dokładne skryptów określony.

Niektóre istotne uwagi:
- Skrypty są zainstalowane pojedynczych plików. W związku z tym tylko jedną kopię skryptu jest zainstalowany, a wiele wersji skryptów nie może być zainstalowane obok siebie w systemie.
- Skrypty może zdefiniować zależności w modułach zewnętrznych, które zostaną zainstalowane po uruchomieniu skryptu instalacji.
- **Zaufany monit:** akceptacji użytkownika jest wymagane w celu zainstalowania skryptów z niezaufanych repozytorium.
- RequiredVersion powoduje zainstalowanie wersji określonej w SxS z istniejącymi wersjami na programu PowerShell w wersji 5.0 lub nowszej.

Symbole wieloznaczne nie są obsługiwane w - nazwie w skrypcie instalacji, Zapisz skrypt i polecenia cmdlet Uninstall-skryptu.

### <a name="scope"></a>Zakres
Określa zakres instalacji skryptu. Dopuszczalne wartości tego parametru to: AllUsers i CurrentUser.

Domyślny zakres instalacji jest AllUsers.

Zakres AllUsers pozwala skryptów można zainstalować w lokalizacji, która jest dostępna dla wszystkich użytkowników komputera, czyli "$env: SystemDrive\Program Files\WindowsPowerShell\scripts".

Zakres CurrentUser umożliwia skryptów można zainstalować tylko na "$home\Documents\WindowsPowerShell\scripts", dzięki czemu skrypt jest dostępna tylko dla bieżącego użytkownika.


Określa zakres instalacji skryptu. Prawidłowe wartości to: AllUsers i CurrentUser. Wartość domyślna to CurrentUser.

Określa zakres AllUsers do zainstalowania skryptu %systemdrive%:\ProgramFiles\WindowsPowerShell\Scripts, dzięki czemu skrypt jest dostępny dla wszystkich użytkowników. Określa zakres CurrentUser instalacji skrypt w $home\Documents\WindowsPowerShell\Scripts, dzięki czemu skrypt jest dostępna tylko dla bieżącego użytkownika.


## <a name="nopathupdate"></a>NoPathUpdate

- Parametr przełącznika NoPathUpdate na temat polecenia cmdlet Install-skryptu Pomija wiersz dodawania lokalizacji instalacji skryptu do zmiennej środowiskowej PATH.
- Użycia polecenia z — NoPathUpdate określony spowoduje żadnego monitu oraz ścieżki nie aktualizowana (Wymuszaj jest do pominięcia tutaj).
- -Force bez — NoPathUpdate spowoduje żadnego monitu i ŚCIEŻKA zostanie zaktualizowany.
- Jeśli podano – Force ani — NoPathUpdate, użytkownik zobaczy monit.
- Wszystko to ma zastosowanie tylko podczas pierwszego skrypt instalacji skryptu jest używany w danym zakresie.


## <a name="notes"></a>Uwagi

To polecenie cmdlet jest uruchamiane w środowisku Windows PowerShell 3.0 ani nowszych wersji środowiska Windows PowerShell w systemie Windows 7 lub Windows 2008 R2 i nowszych wersjach systemu Windows.

Jeśli wartość określona dla parametru Name odpowiada wersji skryptu na komputerze, a parametr MinimumVersion lub RequiredVersion nie zostały dodane, skrypt instalacji dyskretnej będzie kontynuowane bez instalowania tego skryptu. Jeśli zostały podane parametry MinimumVersion lub RequiredVersion i istniejącego skryptu pasuje do wartości tego parametru, następnie wystąpi błąd. Aby dokładniej: Jeśli wersja skryptu aktualnie zainstalowana jest niższa niż wartość parametru MinimumVersion lub nie jest równa wartości parametru RequiredVersion, wystąpi błąd. Jeśli wersja zainstalowanego skryptu jest większa niż wartość parametru MinimumVersion lub równa wartości parametru RequiredVersion, skrypt instalacji dyskretnej będzie kontynuowane bez instalowania tego skryptu.

Skrypt instalacji zwraca błąd, jeśli skrypt nie istnieje w galerii online, który odpowiada nazwie określonej.

Aby zainstalować wiele skryptów, określ tablicę nazw skryptu, oddzielonych przecinkami. Nie można dodać MinimumVersion lub RequiredVersion, jeśli można określić wiele nazw skryptu.

Domyślnie skrypty są instalowane w folderze Program Files. Możesz przekazać wiele obiektów PSGetItemInfo do skryptu instalacji; jest to innym sposobem określania wielu skryptów instalacji za pomocą jednego polecenia.

Które uniemożliwiają uruchamianie skryptów, zawierające złośliwego kodu, zainstalowane skryptów nie są automatycznie importowane przez instalację. Bezpieczeństwa najlepiej, oceny kodu skryptu przed uruchomieniem polecenia cmdlet ani funkcje skryptu po raz pierwszy.

## <a name="install-script-cmdlet-in-pipeline-operations"></a>Polecenia cmdlet skrypt instalacji w operacjach potoku

```powershell

# Find a script and install it
Find-Script -Name "MyDSC*" | Install-Script

# Find a script and install it to the CurrentUser scope
Find-Script -Name "MyDSC*" | Install-Script -Scope CurrentUser

# Find commands by name and install them
# The first command finds the specified commands in the INT repository, and then uses the pipeline operator to pass them to Install-Script to install them.
# The second command uses Get-Installedscript to verify the scripts from the prior command are installed.

# Find multiple role capabilities and install them
Find-RoleCapability -Name MyJeaRole, Maintenance | Install-Script
Get-Installedscript
```

## <a name="install-script-and-get-installedscript-cmdlets"></a>Skrypt instalacji i Get-InstalledScript poleceń cmdlet
Polecenia cmdlet Install-skryptu pozwala zainstalować plik określony skrypt wraz z jego zależności do określonego zakresu. Domyślnie skrypty są instalowane w zakresie AllUsers. Polecenie cmdlet Get-InstalledScript pozwala uzyskać listę plików skryptów, które zostały zainstalowane przy użyciu polecenia cmdlet Install-skryptu.

Użyj Uwaga: Aby umożliwić zarządzanie i lokalizowanie skryptów, gdy są one zainstalowane, skrypt instalacji będzie Utwórz domyślny folder do przechowywania skryptów w $home\Documents\WindowsPowerShell\Scripts i Dodaj tego folderu do środowiska ścieżki. Jeśli modyfikowanie ścieżka jest istotny, zamiast skrypt instalacji skryptu Zapisz. Get-InstalledScripts i skrypt dezinstalacji skryptu może pracować tylko z skrypty umieszczane w systemie za pomocą skryptu instalacji.
